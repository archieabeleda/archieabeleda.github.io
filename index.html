Section 01

What it does

Networks drift. Someone makes a change on a switch at 2am, nobody writes it down, and three weeks later the running config no longer matches what anyone thinks is deployed. Audits get done by hand. The "source of truth" lives in a spreadsheet that went stale months ago.

netops-automation is a small toolkit that attacks that problem from four angles at once. Every tool reads the same device inventory and talks to gear over the same connection layer, so you describe your network once and reuse it everywhere. It's aimed at multi-vendor labs and internal networks where you want change to be tracked, reviewable, and safe to roll back — without buying a heavyweight platform to get there.

Section 02

The toolkit

1

Config backup & drift detection

Pulls the running config from every device and commits it to Git on a schedule. When a config changes, you get a clean diff of exactly what moved — so unplanned changes surface instead of hiding.

does pull configs, commit to git catches untracked changes
2

Pre / post change validation

Snapshots the state that matters (reachability, interfaces, routing, whatever you define) before a change and again after. If something you didn't intend to touch moved, it tells you before you walk away thinking the job's done.

does capture state, compare catches side effects of a change
3

STIG compliance auditing

Checks device configs against security hardening rules and reports what passes and what doesn't. Turns a manual, error-prone audit into something you can run on demand and hand to whoever asks for evidence.

does check hardening rules outputs pass / fail report
4

IPAM source-of-truth sync

Reconciles what's actually on the network with NetBox, so your inventory and IP records reflect reality instead of last quarter's plan. The spreadsheet stops being the thing you don't trust.

does sync devices & IPs to netbox keeps records true
Section 03

How it fits together

The point isn't four scripts. It's four tools sitting on one shared foundation. You define your devices — hostnames, groups, credentials, per-site variables — in one inventory. A single connection layer handles getting onto the gear over SSH, so vendor quirks are handled in one place. Each tool then does its own job on top of that.
